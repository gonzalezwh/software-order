SoftwareForm (js/index.js)

Handles UI events for the Order form, and does a lot of client-side processing to make the order experience smoother for the user.

Summary
SoftwareForm (js/index.js)Handles UI events for the Order form, and does a lot of client-side processing to make the order experience smoother for the user.
Functions
initEssentially a class constructor, gets and formats form data, draws the Order Preview and preps all the jQuery events associated with the UI.
getFormDataPerforms a request to the AJAX enpoint, then processes all the data into locals and the dynamic HTML elements (Department list, Authorizer list, LDAP fields).
displayPreviewPresents a software order preview based on the local data in software_purchase.
autoCompleteResponsePrevents normal autocomplete response so that the input can be post-processed.
updateSoftwareInputDataShows the user the price of the entered software and any notes that may be available.
displayErrorDisplay a tooltip at the location of the dom’s box element.
prepjQueryEventsProxies all UI events so that ‘this’ behavior stays consistent throughout class members.
triggerSoftwareInputIf the user manually types something in (usually on accident), it needs to be processed by updateSoftwareInputData
triggerSoftwareAddThis does some basic error checking of software before adding it to the order.
triggerPreviewRemoveRemoves a row (software) from the order preview.
triggerFormOSIf the OS changes this will delete invalid software from the Order preview.
triggerRequiredWarn the user if a required field is empty.
triggerFormSubmitDoes basic error checking before POST’ing the order.
